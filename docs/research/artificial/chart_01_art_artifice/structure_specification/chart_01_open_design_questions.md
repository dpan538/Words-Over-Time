# Chart 01 Open Design Questions

## Evidence Visibility

- Should `artificial_day` be visible as a small annotation or remain notes-only?
- Should `artificial_memory` be visible despite medium confidence, or should it appear only on hover?
- Should `artificial_lines` remain notes-only even though round 06 confirmed pre-1828 support via `artificiall lines`?
- Should `affected_insincere` be visible in the sense-boundary plane or appear only after hover / interaction?

## Node Hierarchy

- Should `artificer` be a core visible node or a supporting node?
- Should `artifice` be visually emphasized as the bridge node between skill and suspicion?
- Should `artificial` sit on the word-family plane only, or occupy the center of the whole chamber?
- Should weak anchors use smaller node geometry, lower opacity, or both?

## Edge Legibility

- How many edges can remain visible in the full-overlay state?
- Should projection edges use dotted lines while sense-branch edges use solid lines?
- How should the `not_natural -> fake_not_genuine` edge communicate contrast without implying equivalence?
- Should guide labels attach to edges or sit outside the chamber frame?

## Plane Behavior

- Should the sense-boundary plane appear at first load or only after interaction?
- Should the technical plane be secondary / hover-based if the first prototype feels crowded?
- Should each plane have a visible frame, or should the plane be implied by its nodes and projection lines?
- Should the three planes be parallel, or subtly angled for readability?

## Interaction

- Should interaction be scroll-based, hover-based, or click-based?
- Should the camera be fixed, slightly rotatable, or fully locked?
- Should weak anchors appear only in hover labels?
- Should the full-overlay state be automatic at the end of scroll, or user-triggered?

## Static Composition

- Can the chart remain legible as a static screenshot?
- Which labels must remain visible in the static default view?
- Should the chamber include small human-scale marks, or would that pull the reference too close to Soundcube?
- Should the apparatus include axis labels inside the chamber or outside the frame?

## Implementation Readiness

- Should the low-fidelity prototype use SVG/isometric projection first before any 3D scene?
- Should the node inventory stay manually authored, or eventually map to research CSV metadata?
- What is the minimum node set for a readable first prototype?

# Chart 01 Visual Constraints

## Use

- black / white / grey;
- thin linework;
- transparent planes;
- dotted projection lines;
- quiet technical annotation;
- low-motion transitions;
- restrained hover;
- orthographic or mild-perspective camera;
- static-readable default composition;
- deliberate label hierarchy;
- low-opacity weak anchors;
- fixed or tightly constrained camera;
- frame, ticks, and guide marks that behave like notation rather than decoration.

## Avoid

- neon colors;
- cyberpunk interface;
- particle effects;
- AI dashboard style;
- floating random words;
- uncontrolled 3D camera;
- heavy animation;
- decorative 3D network;
- sound-wave simulation;
- copying Soundcube directly;
- free-rotating camera that breaks the reading angle;
- edge bundles that imply equivalence between distinct senses;
- making weak anchors look as important as core evidence.

## Mood

```text
semantic apparatus
technical notation
archival diagram
lexical chamber
quiet spatial reading
```

## Not Mood

```text
sci-fi
AI interface
data dashboard
game environment
word cloud
```

## Static Readability

The default view must work as a still image. If a user never interacts, they should still understand:

- there is a chamber;
- there are three semantic planes;
- `artificial` is central;
- the core direction moves from skill / making toward suspicion;
- `not natural` is not being equated with `fake`.

## Camera Constraint

Use orthographic or mild perspective. The camera may later support small parallax or slight controlled rotation, but the default angle should remain stable and legible.

## Motion Constraint

Motion should reveal layers and paths, not perform spectacle. Avoid continuous ambient motion unless it serves reading.

## Label Constraint

Labels should be sparse, small, and technical. Use guide labels only when they prevent misunderstanding. Weak anchors should appear as hover labels, small annotations, or notes-only content.
